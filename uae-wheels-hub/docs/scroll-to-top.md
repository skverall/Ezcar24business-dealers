# Автоматическая прокрутка к началу страницы

## Проблема
При навигации между страницами в SPA (Single Page Application) страница не прокручивается к началу автоматически. Это особенно заметно при переходе на страницы политик (Cookie Policy, Privacy Policy, Terms of Service) из футера - пользователь остается в том же месте, где нажал ссылку.

## Решение

### 1. Компонент ScrollToTop
Автоматически прокручивает страницу к началу при изменении маршрута.

**Расположение:** `src/components/ScrollToTop.tsx`

**Использование:**
```tsx
import ScrollToTop from './components/ScrollToTop';

// В App.tsx внутри BrowserRouter
<BrowserRouter>
  <ScrollToTop />
  {/* остальные компоненты */}
</BrowserRouter>
```

**Параметры:**
- `smooth?: boolean` - плавная прокрутка (по умолчанию `true`)

### 2. Хук useScrollToTop
Для программной прокрутки к началу страницы.

**Расположение:** `src/hooks/useScrollToTop.ts`

**Использование:**
```tsx
import { useScrollToTop } from '@/hooks/useScrollToTop';

const MyComponent = () => {
  const scrollToTop = useScrollToTop();

  const handleClick = () => {
    scrollToTop({ smooth: true, delay: 100 });
  };

  return (
    <button onClick={handleClick}>
      Наверх
    </button>
  );
};
```

**Параметры:**
- `smooth?: boolean` - плавная прокрутка (по умолчанию `true`)
- `delay?: number` - задержка в миллисекундах (по умолчанию `0`)

## Реализованные улучшения

1. **Автоматическая прокрутка** при переходе между страницами
2. **Плавная анимация** прокрутки для лучшего UX
3. **Кнопка "Наверх"** на длинных страницах (например, Cookie Policy)
4. **Оптимизация производительности** с использованием setTimeout
5. **Гибкие настройки** для различных сценариев использования

## Тестирование

1. Откройте главную страницу
2. Прокрутите вниз к футеру
3. Нажмите на "Cookie Policy"
4. Убедитесь, что страница автоматически прокрутилась к началу
5. Прокрутите вниз и нажмите кнопку "Наверх"
6. Проверьте плавную прокрутку к началу страницы
